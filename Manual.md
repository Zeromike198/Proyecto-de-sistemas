## MANUAL DE BUENAS PRÁCTICAS PARA TRABAJOS UNIVERSITARIOS EN GRUPO

### 1. Introducción
Este manual recopila recomendaciones y lineamientos para que los estudiantes puedan organizar, planificar y ejecutar trabajos universitarios en equipo de manera eficiente. El objetivo es mejorar la comunicación, la coordinación y la presentación de los entregables utilizando buenas prácticas y herramientas adecuadas.

### 2. Organización del equipo
- Definir roles claros (líder, editor, investigador, redactor, presentador).
- Establecer objetivos desde el inicio.
- Crear un calendario de trabajo.
- Acordar horarios de reunión y canales de comunicación.
- Establecer reglas básicas de convivencia y respeto.

### 3. Gestión del tiempo
- Dividir el trabajo en tareas pequeñas.
- Asignar responsables por tarea.
- Establecer fechas límite internas.
- Revisar avances semanalmente.
- Registrar cambios importantes.

### 4. Herramientas colaborativas
- Google Drive y Google Docs para edición colaborativa.
- GitHub para control de versiones.
- Trello o GitHub Projects para gestión de tareas.
- Zoom, Meet o Discord para reuniones.
- WhatsApp o Telegram para comunicación rápida.

### 5. Uso de Git y GitHub en el proyecto
- Crear un repositorio central.
- Trabajar mediante ramas.
- Hacer commits frecuentes y con mensajes descriptivos.
- Revisar aportes mediante pull requests.
- Resolver conflictos de forma colaborativa.
- Documentar versiones en CHANGELOG.md.

### 6. Estructura recomendada de carpetas

/documentos
/imagenes
/referencias


### 7. Normas de citación
- Utilizar formato APA.
- Citar correctamente fuentes bibliográficas.
- Incluir listado de referencias al final del documento.
- Usar gestores como Zotero o Mendeley.

Ejemplos:

Cita textual: “texto citado” (Apellido, año, p. xx)

Referencia: Apellido, Inicial. (Año). Título del libro. Editorial.

### 8. Formato del documento
- Fuente legible.
- Títulos claros y numerados.
- Tabla de contenido.
- Imágenes con pie y referencia.
- Ortografía y gramática revisadas.

### 9. Presentación final
Debe incluir:
- Portada
- Índice
- Introducción
- Desarrollo por secciones
- Conclusiones
- Referencias
- Anexos (opcional)

### 10. Control de versiones del manual
- Registrar cambios en CHANGELOG.md.
- Crear tags en GitHub (v1.0, v1.1, v2.0…).

### 11. Conclusiones
El trabajo en equipo requiere organización, comunicación y disciplina. Aplicar buenas prácticas permite alcanzar mejores resultados y fortalecer la colaboración entre los integrantes.
